# JDBC Connection & Correction Notes

### 📋 Common Mistakes in Original Code:
1. **Duplicate Table Error**: Using `CREATE TABLE student` works the first time, but crashes on every subsequent run because the table already exists.
   * **Fix**: Use `CREATE TABLE IF NOT EXISTS student`.

2. **Resource Leaks**: Manually calling `con.close()` at the end is risky. If an error happens before that line, the connection stays open, wasting memory.
   * **Fix**: Use "Try-with-Resources" blocks to auto-close everything.

3. **Basic Error Reporting**: `System.out.println(e)` provides minimal info.
   * **Fix**: Use `e.printStackTrace()` to see the exact line number and error details.

4. **Missing CRUD Ops**: Creating a table is only half the job. You need to know how to insert and retrieve data safely.
   * **Fix**: Use `PreparedStatement` for inserts and `ResultSet` for reading.

### ✅ Improvements Made:
1. **Try-with-Resources**: Used Java's modern syntax for automatic resource management (Auto-closing).
2. **Prepared Statements**: Implemented professional-grade data insertion to prevent SQL Injection attacks.
3. **Select Query**: Added a loop to fetch and print all records from the database table.
4. **Better Structure**: Organized URL, username, and password into variables for easy configuration.
5. **Interactive Menu**: Added a `while` loop and `Scanner` to create a real application menu where you can choose to Add, View, or Exit.

1. **Compile**:

2. **Run**:
