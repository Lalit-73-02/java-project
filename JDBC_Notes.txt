ADVANCED JAVA NOTES: PART 1 - JDBC
==================================

1. What is JDBC?
   JDBC (Java Database Connectivity) is an API used to connect Java applications with relational databases like MySQL, Oracle, etc.

2. JDBC Architecture
   - JDBC API: Application to JDBC Manager connection.
   - JDBC Driver API: JDBC Manager to Driver connection.

3. Steps to Connect to a Database:
   1. Import packages (java.sql.*)
   2. Register the Driver (Not required since Java 6 if driver auto-loading works)
   3. Open a Connection (DriverManager.getConnection())
   4. Create a Statement/PreparedStatement
   5. Execute the Query (executeQuery or executeUpdate)
   6. Extract data from Result Set
   7. Close the resources

4. Key Interfaces:
   - Connection: Establishes a session with the DB.
   - Statement: Executes static SQL.
   - PreparedStatement: Executes parameterized SQL (Prevents SQL Injection).
   - ResultSet: Holds the query result data.
   - CallableStatement: For stored procedures.

5. Code Example (MySQL):
   -----------------------
   String url = "jdbc:mysql://localhost:3306/mydb";
   String user = "root";
   String pass = "password";

   try (Connection conn = DriverManager.getConnection(url, user, pass)) {
       String query = "SELECT * FROM students WHERE id = ?";
       PreparedStatement pstmt = conn.prepareStatement(query);
       pstmt.setInt(1, 101);
       
       ResultSet rs = pstmt.executeQuery();
       while(rs.next()) {
           System.out.println(rs.getString("name"));
       }
   } catch (SQLException e) {
       e.printStackTrace();
   }

6. Transaction Management:
   - By default, JDBC is in "Auto-Commit" mode.
   - To manage transactions manually:
     conn.setAutoCommit(false);
     // perform operations
     conn.commit(); // if successful
     conn.rollback(); // if error occurs
